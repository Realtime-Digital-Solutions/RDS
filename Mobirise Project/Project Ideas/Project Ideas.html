Medical IoT

Overview

In the ever-growing world of Technology, there are always issues to be solved. One of the key factors in the pursuit of success and gaining traction as a business is by solving problems. Problem solving will always be appreciated in the modern world as consumers and businesses are striving to find ways to make their lives easier.
This brings us to our Project Idea that we believe will compliment an industry that will travel hand in hand with the future, and that is Health. This device will give Doctors/Nurses/Medical personnel the tools to monitor and analyse their patients using a multitude of technology already available to us. By incorporating these established technologies, we believe that we can assist the Healthcare system and do our part in saving people’s lives.

We envision an IoT device that connects to existing Wi-Fi infrastructure in hospitals and in a patient’s home. The data is collected from the IoT device which is then reported to a console. The IoT device will be an ESP8266/Arduino Nano 33IoT that connects to a Local Hotspot or Wi-Fi connection, which then will be transmitted via Wi-Fi. The information will be displayed on a GUI (Graphical User Interface) in the form of a Web page. A PC application and smartphone and tablet support could be added in future updates. This device will display the data collected and report the position of the IoT device relative to a Wi-Fi hotspot or a Wi-Fi connection point using RSSI (Receiver Signal Strength Indicator) levels to determine position. GPS would be another alternative for position location however buildings and underground carparks do not always offer GPS signals as well as pose great difficulty. This is also highly costly. The data collected from the IoT device could be heart rate from a patient, relative position, and vital signs as well as position of a Nurse/Doctor in a hospital or position of vehicle in an underground carpark.

Description

The IoT heartrate monitor/tracker consists of an IoT device that is battery operated and is worn by the patient. It will use an ESP8266 or an Arduino Nano 33 IoT unit that will monitor the patient’s heartbeat via a sensor that detects light through the skin. This sensor uses infrared to achieve this. The sensor will require 3.3V to 5V for its voltage supply and this will be supported by the Arduino Nano or ESP8266. The IoT device will run from a 9V Alkaline battery or 3.6V lithium-ion battery and the internal regulators will supply the 5V. The communication from the heartbeat sensor to the IoT microcontroller will use I2C Serial Interface. This serial interface will provide the microcontroller with the data from the heartbeat sensor. This data will be analysed, processed, and then sent out via Wi-Fi (TCP/IP/UDP) to the monitoring software. The MAC address of the IoT device will aid in the individual identification of the device.

The Wi-Fi connection on the PC Laptop/Desktop will be receiving the data from the IoT device. The monitoring software will be constantly taking data from the IoT device and displaying the data on its GUI interface as beats per minute (bpm). It will also be monitoring the receiver strength signal (RSSI) of the IoT device to allow it to display an approximate location to the receiving laptop/desktop. This will be displayed on a simple map at first and then progress to something more complex in future developments. The position of the sensor will rely heavily on the Wi-Fi hotspot’s ability to receive quality radio signals. If there are enough Wi-Fi hot-spots, a process of triangulation will be employed for the accuracy of location. There will be a maximum and minimum setting for the heart-rate data that will produce an alarm if the heart-rate is above or below the set limits. This is to alert a Healthcare Professional that the patient is suffering and may require urgent medical attention. The monitoring software will be written in Python 3.x and will use socket programming to interface to the Wi-Fi IoT device. The IoT device will use C/C++ to program its firmware.

A web page with similar information of patient statistics can also be developed. This web page can be hosted on a local server or cloud and storage of vital data such as time, position, heartrate etc. For future data analysis, development of Ethernet network connection to the monitoring software will also be implemented. Development of over the air configuration of the IoT device for quick and easy deployment as the device is changed from one patient to another. This feature will be added to the existing applications and will require special permission such as a password to program the device. Security of the data will also be implemented as to protect the patient’s details and to help eliminate data corruption. 

Tools & Tech

Tools and Software/Firmware requirements

There are several software tools, hardware and programming languages that are required for our IoT project.

Our primary goal:

• Develop an application in Python
• Develop an IoT device to monitor patient vital signs
• Display the data from the IoT device on the PC application
• Use Wi-Fi hotspots/existing Wi-Fi infrastructure
• Display the position of the IoT device relative to Wi-Fi hotspots
Software/Firmware Tools (Free and open sourced):
• JetBrains Python IDE, Community PyCharm, VS Code or equivalent
• Arduino IDE
• QT designer

In our IoT project we will utilize JetBrains Community PyCharm Python IDE as it is open sourced, free and provides linting to assist in proper Python programming syntax and generates clean and neat code. PyCharm has been specifically designed for the Python programming language, so it offers the best functionality and ease of use for our Python program application.

The Arduino IDE will be the best development environment as it supports the Nano 33 IoT board and IoT Wi-Fi development. There are examples of code that we can gain inspiration from for our design in the Arduino IDE and help motivate us. The Arduino libraries for Wi-Fi and the Nano 33 IoT board will help reduce the amount of code that we have to write. The Arduino community is vast and comprehensive. If we experience difficulties in our code development, the Arduino community will be able to help resolve our coding issues. There is a possibility of using the Atmel Studio IDE, however we may not have the support nor the libraries that will help us achieve our programming goals.

Our decision to use QT designer as a drop and drag GUI application developer was for the ease of use and to develop a GUI application without the complexity of learning a new framework module in Python. There is plenty of support for this software tool as well.

Hardware required:

1. ESP8266 or Arduino Nano 33 IoT (Digikey)
2. PC, Laptop or Desktop computer
3. Heart rate detection device XC3740 heartbeat module (Jaycar)

The Arduino Nano 33 IoT offers everything that is needed in terms of an IoT device, from Ublox Wi-Fi module, serial interfaces, 48Mhz 32-bit ARM microcontroller, crypto IC, and 3 axis accelerometers. It is relatively easy to program via the Arduino IDE and readily available. The Nano 33 IoT will be powered by an alkaline 9-volt or 3.6-volt lithium-ion battery and the XC3740 heartbeat monitor will be powered via the 3V3 output of the Nano 33 IoT. The serial communication from the XC3740 will be I2C (Inter-Integrated circuit) and will parse its data through this interface. (see figure 1 below)
The XC3740 heartrate module is simple to interface on the Nano 33 IoT using I2C, is low cost, and can be purchased from any local electronics component supplier store. 

Figure 1: Basic schematic of heart-rate monitor connection to Arduino nano 33 IoT

Programming languages:

• Python 3.6 or above.
• C.

The choice to use the Python programming language was simple. We have some experience with this general-purpose programming language and is becoming popular in the programming realm.
The connection to the Wi-Fi, from the Nano device will consists of a server/client relationship. The PC that will run the Python interface application, will use a basic socket server and bind to the Wi-Fi interface. The Python application is to be wrapped so it can be run as an executable file.
The Arduino platform requires the use of the C programming language to program the Nano 33 IoT. Our team has some experience with this programming language. The Nano 33 IoT will serve as a client to the Python application and data will be extracted with the application and displayed on the GUI as heartrate (BPM) and position/RSSI (receiver signal strength indication).
The ESP8266 and Arduino Nano 33 IoT are Wi-Fi capable devices with on board microcontrollers that can interface with many different hardware sensors including the I2C serial interface. The IoT device will use the open-sourced Arduino IDE platform for programming and will use C or C++ for the programming language. The PC application will use the Python 3.x interpreter. The application will be an executable file and developed in an IDE such as Community PyCharm or equivalent, which is an open-source development environment. 

Skills Required

Basic knowledge of serial data, electronics, microcontrollers, and Wi-Fi radio propagation. To write code for the Arduino or ESP8266 it will require reasonable knowledge of C/C++ and the Arduino IDE platform. There will also be a need to have a basic understanding of TCP/IP (Transmission Control Protocol/Internet Protocol), and UDP (User Datagram Protocol). The PC application will require reasonable knowledge of Python, PyCharm/VS Code, and socket programming. The GUI (Graphical User Interface will use TKinter or QT for development. Page (TKinter) and QT Designer are drag and drop GUI designers that will help with the look and feel of the application for the PC. The software for both is free, open-sourced and require some time to learn. PyGame, which is a Python framework, maybe used to design the IoT device location map.
