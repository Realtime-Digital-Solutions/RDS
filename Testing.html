Testing

To test the Arduino and heart rate monitor, our lead software developer has created a program in Python that uses the Arduino to relay an output data stream coming from the heart rate monitor. Using the heart rate monitor attached to a person, we will be able to get physical heart rate data. To develop and refine the heart rate detection capabilities of our program, we will require willing participants, or team members, to participate in relaxation and energetic exercises while wearing the device. These studies will help to test the program’s heart rate rhythm detection that will trigger program events, to notify the GUI of rhythm stability issues.

When testing with participants is not viable or the hardware devices are not available to work on, we will use a modified version of the Arduino testing program that can be used on a computer, more specifically, for Windows OS. This program replaces the physical heart rate output stream with a software dummy stream. This output data stream will be configurable from within the testing GUI and allow us to test different heart rate rhythms. We will use the data provided from the dummy stream to test and refine our program’s ability to detect abnormal heartbeat patterns. We will also rely on heart rate medical research to help define relevant heart rate rhythms that will trigger program events.

We will use the modified testing program to develop and test the main functions in our GUI. In the early stages of development, this will enable us to develop, define and test our GUI in a PC environment without requiring the Arduino and heart rate monitor devices. This GUI will also be used to identify if the application is correctly initiating program events that provide notifications to the GUI regarding heart rate rhythm and “Man Down” detection.

Further along development, we will need to test implementation of the accelerometer “Man Down” feature. We will need to conduct safe drop tests of the Arduino device and test participants wearing the device who will conduct a series of actions falling onto a mattress. During these activities, we will be monitoring the data collected from the accelerometer to determine parameters and conditions that indicate a fall.

Once our project becomes more complete and functional, we would like to implement location tracking within the home. For the team to develop, implement and test this functionality, this will require the use of 3 or more WI-FI access points set up within a room, and the Arduino device. These tools will be used to test WIFI beacon triangulation to track the Arduino’s location within the room. We will also need to investigate tools and technologies to make a scale floor plan of the room and test how the location tracking is mapped to the floor plan. As the home location tracking feature requires further research, tools, technologies, a complex installation process and extended GUI development, we have decided to explore this feature later into a future development.
